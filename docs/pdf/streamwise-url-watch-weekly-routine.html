StreamWise URL watch — weekly routine

Beginner checklist · matches GitHub Action “URL watch” · repo file .github/workflows/url-watch.yml

What this is for

The workflow checks official pages listed in data/url-watch-list.json. If a page’s HTML fingerprint changes, GitHub opens an Issue so you can re-verify pricing on the real site.

It does not automatically update prices in StreamWise.

Step A — Monday morning (about 5 minutes)

  1. Open your GitHub repository in a browser.
  2. Click Actions.
  3. In the left sidebar, click URL watch (not CI).
  4. Click Run workflow → choose branch main → click the green Run workflow button.
  5. Wait until the run shows a green checkmark (Success).

If it fails (red X): open the run → click the job → scroll to the bottom → read the error.

Step B — decide if you need to do anything (about 2 minutes)

B1) Check GitHub Issues

  1. Click Issues.
  2. Look for a new issue titled like URL watch: monitored pricing pages changed.

If there is no new issue, you’re done for the week (no fingerprint changes vs the saved baseline).

B2) Check commits (optional)

Sometimes the workflow commits an update to data/url-watch-baseline.json. That is normal: it stores the new “normal” fingerprint so you are not alerted again until the next change.

When you get an issue — outcomes 1 to 3

Open the issue. It lists URLs and old/new fingerprints.

Outcome 1 — False alarm

Meaning: the page changed, but not necessarily the price.

Examples: banners, layout, marketing copy, cookie text.

What you do:

  1. Open each URL in a browser (incognito is fine).
  2. Confirm pricing/eligibility still matches your catalog.
  3. Close the issue with a short comment: Verified: no pricing change.

Catalog update: none.

Outcome 2 — Real change

Meaning: the official page shows a different bundle, price, or eligibility than your catalog.

What you do:

  1. Update your Google Sheet (or JSON if that’s your workflow).
  2. Re-download CSVs into data/catalog-sheet-export/services.csv and options.csv.
  3. Run apply + diff:
cd /Users/chrisanderson/streamwise
npm run catalog:sheet:apply -- catalog-before.json data/catalog-sheet-export/services.csv data/catalog-sheet-export/options.csv catalog-merged.json
npm run catalog:diff -- catalog-before.json catalog-merged.json
  1. If the diff looks correct, publish (token from Vercel; use read -s CATALOG_ADMIN_TOKEN so it pastes safely): 
read -s CATALOG_ADMIN_TOKEN
curl -sS -X PUT "https://streamwise-xi.vercel.app/api/catalog" \
  -H "Content-Type: application/json" \
  -H "x-admin-token: $CATALOG_ADMIN_TOKEN" \
  --data-binary @catalog-merged.json
  1. Hard refresh the live site: Cmd+Shift+R.
  2. Close the issue: Updated catalog after verified pricing change.

Outcome 3 — Blocked / too noisy

Meaning: GitHub can’t fetch reliably, or the page changes constantly for non-pricing reasons.

What you do:

  1. In data/url-watch-list.json, set "skipAutomation": true for that row.
  2. Commit and push to main.
  3. Use Distill (or similar) on your home internet for that URL.
Note: your live GET /api/catalog response can be cached for up to about an hour after publishing.
